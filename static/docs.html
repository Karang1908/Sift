what is sift

Sift is a single-screen document analysis dashboard. A small, fixed set of hardcoded accounts log in, upload documents (PDF, DOCX, XLSX, PPTX, images, text, RTF, ODT), get them parsed to text server-side, run AI-assisted analysis queries against a local Ollama instance, and export the results as formatted Excel, PDF, or Word documents.

Every account's data is filesystem-isolated from every other account's. An is_admin-flagged account can see a permanent, append-only audit trail of everyone's activity through a separate admin panel.

It runs as one Python process on one machine, reachable either only from that machine (127.0.0.1) or from every device on the local network (0.0.0.0). There is no cloud backend, no container orchestration, no database server — persistence is flat files on the host's own disk.

Stack: FastAPI + Uvicorn (Python 3.13+), vanilla JS frontend, local Ollama daemon proxying to minimax-m3:cloud. Deployment model: single-process, single-host, LAN-shared, no database, no cloud hosting.

features

📄

multi-format parsing

Upload PDF, DOCX, XLSX, PPTX, images (via OCR), RTF, ODT, and plain text. All files are parsed server-side into cached text for instant analysis.

🤖

ai-powered analysis

Run natural-language queries against your uploaded documents via a local Ollama instance. Results stream in real-time via Server-Sent Events.

📊

formatted export

Export analysis results to Excel, PDF, or Word. Two pipelines: an AI script generator and a template-based clone pipeline for precise formatting.

🔒

per-user isolation

Every user's uploads, caches, presets, and templates live in their own filesystem namespace. No shared pools, no cross-user data leaks.

📋

permanent audit trail

Every upload, analysis, and export is archived permanently. Deleting from the workspace never touches the archive. Admins see everything.

✨

prompt enhancement

Refine your analysis prompts and export instructions with one-click AI rewriting before running them — better input, better output.

💾

presets system

Save and reuse frequently-used analysis prompts and export instructions as named presets. Separate systems for analysis and export workflows.

🖥️

one-click launcher

Double-click start_mac.command or start_windows.bat. Handles Ollama, dependencies, the server, and opens your browser.

component architecture

Everything inside "Host Machine" is one OS process — there's no internal network hop between the web layer, the parser, the audit log, and the filesystem; those are plain Python function calls. The only two real network hops are Browser↔Server (LAN, plaintext HTTP) and Server↔Ollama (localhost, then out to the internet for :cloud inference).

CORSMiddleware uses allow_origins=["*"] but no allow_credentials=True. Intentional — a wildcard with credentials would let any site ride the session cookie cross-origin; without credentials, the wildcard is inert.

The session TTL is fixed from login time, not sliding — a user active at hour 5:59 is still logged out at hour 6:00. The in-memory store means the entire team gets logged out simultaneously any time the process restarts, including a uvicorn --reload triggered by editing any watched file.

per-user data isolation

Every account's data lives under its own subdirectory, never a shared pool. username is only ever a value from get_current_user() — a validated USERS key — so joining it into a path carries no traversal risk.

Two independent isolation mechanisms are stacked:

  1. Live workspace (uploads/, parsed_cache/, presets/, export_templates/, export_presets/) — reflects what a user currently sees. Deleting a file removes it from here.
  2. Permanent archive (audit_uploads/, audit_analysis/, audit_exports/) — written once at creation time, never modified. logs/activity.jsonl is the one shared file — the admin's cross-user activity feed.

request flow: file upload

sequenceDiagram
    participant B as Browser
    participant S as app.py: upload_file()
    participant P as parser_utils
    participant FS as "uploads/ · parsed_cache/"
    participant AL as audit_log.py

    B->>S: POST /api/upload (multipart file)
    S->>S: safe_filename = basename, reject video extensions
    S->>FS: write original bytes to uploads/[user]/
    S->>P: extract_text_from_file(filepath)
    Note over P: dispatches by extension:
PDF/DOCX/XLSX/PPTX native parsers,
images via OCR, legacy formats via
macOS textutil, binary sniff fallback
    P-->>S: parsed text (or Error sentinel)
    S->>FS: write .txt to parsed_cache/[user]/
    S->>AL: archive_upload_copy()
    S->>AL: log_activity(user, "upload", ...)
    S-->>B: 200 (filename, size, status)
The permanent archive copy is made at upload time, independent of whatever happens next — even a delete two seconds later can't lose it.

request flow: analysis query (sse)

sequenceDiagram
    participant B as Browser
    participant S as app.py: process_files()
    participant O as Ollama daemon
    participant AL as audit_log.py

    B->>S: POST /api/process (prompt)
    S->>S: concatenate every .txt in user's parsed_cache
    S->>O: POST /api/chat (model, messages, stream:true)
    activate S
    loop token stream
        O-->>S: message content token
        S-->>B: SSE: content token
        Note over S: accumulate content in memory
    end
    O-->>S: done: true
    S-->>B: SSE: done: true
    deactivate S
    S->>AL: asyncio.shield(persist):
archive_analysis() then log_activity()
    Note over AL: shielded so a client disconnect
at stream-end can't cancel the write
Why the shield matters: without asyncio.shield(), a client disconnecting at stream-end can cancel the two-step archive-then-log sequence, leaving an archived .md with no matching activity-log entry.

request flow: document export

Two pipelines handle export. The choice depends on whether a template is provided and whether it has fillable fields.

    subgraph ScriptPipeline["Script pipeline"]
        direction TB
        SP1["Model writes a Python<br/>script for this report"]
        SP2["AST allowlist validation"]
        SP3["Run in subprocess<br/>with CPU + wall-clock limits"]
        SP4{"Script succeeded?"}
        SP5["Deterministic fallback"]
        SP1 --> SP2 --> SP3 --> SP4

    end
    subgraph ClonePipeline["Clone pipeline"]
        direction TB
        CP1["Extract template structure<br/>cells / paragraphs / form fields"]
        CP2["Model returns JSON<br/>field-mapping only"]
        CP3["Trusted splice function<br/>applies mapping to copy"]
        CP1 --> CP2 --> CP3
    end
    SP4 -->|Yes| FileBytes
    SP5 --> FileBytes
    CP3 --> FileBytes["file_bytes in memory"]
    FileBytes --> ArchiveStep["archive_export()<br/>shielded persist"]
    ArchiveStep --> SSE["SSE: base64 file"]
    SSE --> Done(["Browser downloads"])

The clone pipeline is the smaller attack surface: the model never writes or runs code — it only returns a bounded JSON object that a trusted function applies. The script pipeline does run LLM-generated Python on the host, guarded by an AST import/name allowlist and subprocess resource limits. Defense-in-depth, not a real sandbox.

security posture

concernstatusdetail
Password storage✅ goodbcrypt hashes only, never plaintext, constant-time comparison
Brute-force✅ basic5 failures → 60s per-username lockout
Session token✅ goodsecrets.token_urlsafe(32), HttpOnly, SameSite=Lax
Data isolation✅ structuralFilesystem-namespaced by username
Transport❌ nonePlain HTTP — credentials sniffable on the LAN
CORS✅ hardenedWildcard origin, no allow_credentials
Admin authz✅ explicitget_current_admin 403s non-admin accounts
Path traversal✅ guardedos.path.basename() + regex validation
LLM code exec⚠️ defense-in-depthAST allowlist + subprocess rlimits, not a real sandbox
API discovery⚠️ minor/docs and /redoc reachable without login
Audit integrity✅ verifiedasyncio.shield() survives client disconnects
Retentionℹ️ by designArchive grows forever — no pruning policy yet
Bottom line: this is correctly scoped as a trusted-LAN, known-small-team tool. The single biggest change for external exposure would be transport security — put a reverse proxy with a TLS certificate in front of it.
